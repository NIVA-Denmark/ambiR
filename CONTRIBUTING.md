# Contributing to ambiR

This is how you can contribute to ambiR. The authors are happy to
receive all forms of feedback so that we can maintain and improve the
package.

### Code of Conduct

By *participating*, that is by contributing to, or commenting on the
ambiR *project*, you agree to abide by the [code of
conduct](https://niva-denmark.github.io/ambiR/CODE_OF_CONDUCT.md).

### Corrections to species list

This concerns errors in the species lists. Examples could include
misspelling of a species name or allocation of a species to an incorrect
AMBI group.

*If you find such an issue, what should you do?*

The preferred place to register such an issue is in the ambiR GitHub
repository. You should first check if the issue has already been raised
by someone else.

[https://github.com/NIVA-Denmark/ambiR/issues?q=label:“Species%20data”](https://github.com/NIVA-Denmark/ambiR/issues?q=label%3A%22Species%20data%22)

You do not need to be a registered GitHub user to view existing issues.
If you want so submit a new issue, you will need to register as a user.

Please remember to apply the **Species data** label when submitting the
issue.

If you do not want to register with GitHub, please contact **aborja *at*
azti.es** or **cjm *at* niva.dk**. But, do *please* check first if the
problem has already been noted.

Corrections will be gathered and used to update future versions of the
species list. Remember that, while you are waiting for updated versions,
you are free to use your own user-defined species lists, including any
corrections you deem necessary.

If you are a GitHub user, you can also use the *issue* functionality to
record issues.
